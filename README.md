## 📌 Project Overview

This project was developed as part of my **Data Technician Bootcamp**, focusing on building a cloud-based data engineering solution using Microsoft Azure.

The goal was to simulate a real-world data pipeline where raw data is ingested, processed, stored, and prepared for analytics using modern cloud services.

It reflects common enterprise workflows used in business intelligence, data engineering, and cloud analytics environments.

---

## 🎯 Project Objectives

- Move raw data into a cloud environment  
- Automate data ingestion and transformation workflows  
- Build scalable storage and analytics architecture  
- Enable reporting and analysis through structured datasets  
- Apply security and governance best practices  

---

## ☁️ Cloud Architecture Used

This project integrates multiple Azure services:

- 🗄️ Azure Blob Storage — raw and processed data storage  
- 🧊 Azure Data Lake Gen2 — scalable analytics storage  
- 🗃️ Azure SQL Database — structured relational data  
- 🔄 Azure Data Factory — ETL pipeline orchestration  
- 📊 Power BI / Synapse Analytics — reporting and insights  

---

## 🔄 Data Pipeline (ADF)

Using Azure Data Factory, I built automated pipelines to:

- Ingest data from source systems  
- Transform and clean datasets  
- Load data into structured and analytical storage layers  
- Schedule and automate workflow execution  

Key ADF components used:

- Linked Services  
- Datasets  
- Pipelines  
- Triggers  
- Activities (Copy, Transform, Load)

---

## 🧹 Data Engineering & Storage Design

### Data Formats Used:
- CSV — raw ingestion layer  
- JSON — semi-structured data sources  
- Parquet — optimized analytical storage  

### Data Modelling Approach:
- Designed relational structures using keys and relationships  
- Reduced redundancy through structured schema design  
- Created curated layers for analytics readiness  

---

## 🧠 SQL & Data Analysis

Using Azure SQL Database and AdventureWorks-style datasets, I performed:

- Filtering and segmentation of customer data  
- Product and pricing analysis  
- Sales aggregation and reporting queries  
- Multi-table joins for relational insights  

SQL was also used to validate pipeline outputs and ensure data accuracy.

---

## 🔐 Security & Governance

This project followed industry-standard data protection practices:

- GDPR compliance principles applied  
- Data minimisation and controlled access considered  
- Role-Based Access Control (RBAC) explored  
- Secure storage and encrypted data transfer reviewed  

Additional focus areas:

- Data Protection Act (2018) awareness  
- Secure handling of sensitive datasets  
- Governance aligned with enterprise cloud standards  

---

## 🛡️ Reliability & Backup Strategy

To support resilience and continuity:

- Azure Backup used for data snapshots  
- Geo-redundant storage (GRS) considered for durability  
- Disaster recovery concepts explored using Azure Site Recovery  

---

## 📊 Analytics & Reporting

Data prepared in Azure was designed for downstream analytics using:

- Power BI dashboards  
- Azure Synapse Analytics  
- Curated data lake layers  

Example insights supported:

- Sales performance trends  
- Customer segmentation analysis  
- Product and inventory behaviour  
- Operational reporting views
